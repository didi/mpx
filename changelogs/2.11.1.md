# v2.11.1

## ✨ New Features
- **跨端输出 React Native**
  - 新增 UnoCSS 原子类支持，提供 RN 专用 preset 与构建链路，覆盖常用排版、Flex、间距、背景与渐变、边框、阴影、Transform、Transition 等工具类，并在编译阶段拦截不受支持的工具类与 variants。
  - 完善 RN 原子类运行时样式合并，支持 `!important`、preflight / App / UnoCSS / 普通样式分层优先级、媒体查询、主题与方向 variants、`hover`、Directives、Variant Groups 及 CSS 变量颜色透明度处理。
  - 将 UnoCSS 升级至 66，并将 `@mpxjs/unocss-base` 切换为基于 `preset-wind3` 的 ESM 配置方式。
  - 扩展 `@mpxjs/perf` 性能探针：`mark(name)` 支持记录有序时间线，新增 `measureStart(name)` / `measureEnd(name)` 跨作用域耗时 API，Reporter 可接收 `MarkTimeline`，默认 Console Reporter 同时输出聚合耗时与时间线。
  - 性能探针时间线自动包含 `start` / `end` 边界并限制为 256 条事件；旧 `mark` / `measure` 耗时写法需迁移至 `measureStart` / `measureEnd`，旧 `measure` 不再导出。

- **Webview Bridge**
  - 新增 Touch 'n Go App 内小程序环境识别，H5 页面可复用支付宝小程序的 Webview Bridge 能力。

## 🐛 Bug Fixes
- **跨端输出 React Native**
  - 修复 iOS、Android 与 Harmony 环境中非通用事件使用 `catch` / `capture-bind` / `capture-catch` 时可能无法触发的问题；编译器现会给出警告并降级为普通 `bind`，同时保留通用触摸事件的冒泡与捕获能力，以及 `movable-view` 的 `catchhtouchmove` / `catchvtouchmove` 能力。
  - 修复 RN 原子类中 `box-content` 与受支持的 Transition 工具类被误拦截，以及部分 aria / data variants 校验异常的问题。

- **构建与运行**
  - 修复异步分包 `componentPlaceholder` 资源解析结果缺失时可能触发的构建异常。
